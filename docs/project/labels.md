# Suggested GitHub labels

Use labels to keep the backlog searchable and to enforce the Community/Pro split.

## Type
- `type/feature`
- `type/bug`
- `type/docs`
- `type/chore`

## Tier
- `tier/community`
- `tier/pro`

## Priority
- `prio/P0` (blocker / critical)
- `prio/P1` (high)
- `prio/P2` (medium)
- `prio/P3` (nice to have)

## Areas
- `area/tls`
- `area/acme`
- `area/http`
- `area/static`
- `area/proxy`
- `area/config`
- `area/observability`
- `area/security`
- `area/systemd`
- `area/docs`
- `area/wasm`
- `area/control-plane`
- `area/fleet`

## Milestones (suggested)
- `v0.2` (single-node prod-ready)
- `v0.3` (operator UX + reload)
- `v1.0` (complete community server)
- `pro-v1` (fleet + advanced security/obs)
